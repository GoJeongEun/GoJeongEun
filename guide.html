📝css 메모장

💠 place-content = align-content, justify-content 합친 css의 약어
- 브라우저 호환성 체크()
💠 background-position의 경우 여러개를 한꺼번에 제어 가능하다 (조건: background 한꺼번에 적용 시)
ex> top right 15%, bottom left 12%, bottom center 콤마를 기준으로 하나씩 줄 수 있음
💠 태그 안에 data-* 속성을 사용해서 변수처럼 활용 가능 & css도 적용가능
ex> a태그 안에 data-name="test" 로 줬을 때 사용법
 📁 css파일에 [data-name=test]{dispaly:none}
 data-* 타입 여러개 중복으로 사용 가능

📝 디지털 정부서비스 UI/UX 가이드

Step1. Color

  1. 01. Primary
    Primary Color 버튼, 활성 상태와 같은 UI 전체의 주요 구성 요소에 사용된다.
  2. 02. Secondary color
    Secondary color 필터, 칩과 같이 UI에서 보조적인 역할을 하는 구성 요소 전반에 사용된다.
  3. 03. Grayscale
    Grayscale은 정보전달과 위계를 나타내는 중요한 역할을 한다. 주로 BG, Text, line에 사용되며 특히 텍스트는 가독성을 고려해야 하기 때문에 배경과의 명암비를 고려해야 한다.
  4. 04. Alpha
    다양한 배경 색상과 콘텐츠
  5. 필요한 서류 & 확인해야 되는 것들
    🎈 종류 : hug, hf

    🕒 실행 기간 (1달)
    - hf(심사 빠름) : 신용기준
    - hug(심사 느림)
    📝1. 가심사 필요 서류 - 가계약 영수증 (5%) - 인감 -

    📝2. 가심사 완료 후 지정 은행방문 - 밑에 서류

    📌 확인할 것 - 등기부등본(인터넷등기소 - 유료)
    - 보험 가입 여부
    - 강아지
    - 해당 집 대출여부

    📝 서류 - 소득증명서
    - 재직증명서
    - 건강보험가입내역서
    - 인감
